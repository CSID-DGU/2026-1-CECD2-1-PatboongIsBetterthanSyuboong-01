## 🔴 심각한 문제점

### 1. **스코어 분포 문제 (가장 심각)**

**증상**:
- Threshold가 거의 0 (4.777e-10)
- 모든 샘플의 스코어가 거의 0에 가까운 값으로 수렴
- F1-optimal threshold가 너무 작아서 거의 모든 것을 이상으로 분류

**원인 분석**:
1. **ADRM2D의 비선형 remapping이 스코어를 0으로 수렴시킴**
   - `x1 = s + a * (s^2 - s)` 공식에서 `s^2 - s` 항이 작은 s 값에서 음수가 될 수 있음
   - `alpha`와 `beta`가 학습되지 않아 항상 0에 가까운 값일 가능성
   - Outlier suppression이 너무 강하게 작용하여 모든 값을 0으로 만듦

2. **Stability Penalty가 스코어를 0으로 밀어냄**
   - `stability_target=0.05`로 설정되어 있어 스코어 평균을 0.05로 수렴시키려 함
   - 하지만 실제로는 0에 가까운 값으로 수렴

3. **스코어 정규화 문제**
   - `s_recon = 1.0 - torch.exp(-se)`에서 `se`가 매우 작으면 `s_recon`도 거의 0
   - ADRM2D를 거치면서 더욱 작아짐

### 2. **모델 학습 실패**

**증상**:
- ROC-AUC가 0.5 근처 (랜덤 수준)
- 모든 클래스의 ROC-AUC가 0.48~0.52 사이
- 모델이 정상과 이상을 전혀 구분하지 못함

**원인 분석**:
1. **학습 데이터 불균형**
   - 정상: 1362개, 이상: 11545개 (약 8.5:1)
   - 하지만 학습 시에는 정상 데이터만 사용하므로 이 문제는 아님

2. **Loss 함수 문제**
   - `loss_score = score.mean()`이 정상 데이터에서 스코어를 0으로 만드려 하지만
   - 실제로는 스코어가 이미 0에 가까워서 학습이 안 됨

3. **Gradient Vanishing**
   - 스코어가 거의 0이면 gradient도 거의 0
   - 모델이 학습되지 않음

### 3. **Confusion Matrix 분석**

```
        예측
실제    정상  이상
정상    1218   144
이상    9879  1666
```

**문제점**:
- **FN (False Negative)**: 9879개 - 이상의 85% 이상을 정상으로 잘못 분류
- **TP (True Positive)**: 1666개 - 이상 중 14%만 제대로 탐지
- **Threshold가 너무 작아서** 거의 모든 것을 이상으로 분류하려 하지만, 실제로는 대부분을 정상으로 분류함
- 이는 **스코어가 거의 0이어서** threshold와 비교할 때 문제가 발생

### 4. **데이터 불균형 문제**

**평가 데이터**:
- 정상: 1362개 (10.5%)
- 이상: 11545개 (89.5%)

**영향**:
- PR-AUC가 높게 나오지만 이는 데이터 불균형 때문
- 실제 성능은 ROC-AUC가 더 정확한 지표
- Balanced evaluation을 사용했지만 여전히 문제가 있음

## 🔧 해결 방안

### 1. **스코어 정규화 개선 (최우선)**

**문제**: 스코어가 0에 수렴하는 문제

**해결책**:
1. **ADRM2D 수정**:
   ```python
   # 현재: x1 = s + a * (s^2 - s)
   # 개선: 스코어가 너무 작아지지 않도록 보정
   x1 = s + a * (s * (1 - s))  # s가 0에 가까우면 항이 작아짐
   # 또는
   x1 = torch.clamp(s + a * (s^2 - s), min=0.01, max=0.99)
   ```

2. **Stability Penalty 조정**:
   - `stability_target`을 0.05가 아닌 0.1~0.2로 증가
   - 또는 스코어가 너무 작아지지 않도록 penalty 수정

3. **스코어 스케일링**:
   ```python
   # ADRM2D 출력 후 스코어를 재스케일링
   score = (score - score.min()) / (score.max() - score.min() + 1e-8)
   ```

### 2. **Loss 함수 개선**

**문제**: 현재 loss가 스코어를 0으로 만드려 함

**해결책**:
1. **Score loss 수정**:
   ```python
   # 현재: loss_score = score.mean()
   # 개선: 정상 데이터는 낮은 스코어, 하지만 0이 아닌 적절한 값
   target_score = 0.1  # 정상 데이터의 목표 스코어
   loss_score = F.mse_loss(score.mean(), torch.tensor(target_score))
   ```

2. **Reconstruction loss와 Score loss 균형**:
   - 현재: `loss = loss_recon + 0.1 * loss_score`
   - 개선: `loss = loss_recon + 0.5 * loss_score` (score loss 비중 증가)

### 3. **학습 전략 개선**

**문제**: 모델이 학습되지 않음

**해결책**:
1. **학습률 조정**:
   - 현재 learning rate 확인 필요
   - Alpha/Beta head의 learning rate를 더 크게 설정

2. **학습 단계 분리**:
   - 1단계: AE만 학습 (reconstruction)
   - 2단계: AE 고정, Attention branch 학습
   - 3단계: 전체 모델 fine-tuning

3. **Gradient Clipping 강화**:
   - 현재: `max_norm=1.0`
   - 개선: `max_norm=0.5` 또는 더 작은 값

### 4. **데이터 처리 개선**

**문제**: 평가 데이터 불균형

**해결책**:
1. **Balanced evaluation 강화**:
   - 평가 시 정상/이상 비율을 1:1로 맞춤
   - 이미 구현되어 있지만 제대로 작동하는지 확인 필요

2. **FPR 기반 threshold 사용**:
   - F1-optimal 대신 FPR 기반 threshold 사용
   - 예: `--target_fpr 0.05` (FPR 5%로 threshold 설정)

### 5. **모델 구조 개선**

**문제**: ADRM2D가 스코어를 0으로 만듦

**해결책**:
1. **ADRM2D 비활성화 테스트**:
   - ADRM2D 없이 단순 곱셈으로 결합: `score = s_recon * s_feat`
   - 성능 비교

2. **Fusion 모듈 활성화**:
   - 현재 `use_fusion=False`로 되어 있음
   - `use_fusion=True`로 변경하여 테스트

3. **Alpha/Beta 초기화**:
   - 현재 랜덤 초기화
   - 0이 아닌 작은 양수로 초기화
