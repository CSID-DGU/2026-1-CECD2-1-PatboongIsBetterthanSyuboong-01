# 선행 특허 조사 결과

## 요약

본 문서는 흉부 X-ray 이상 탐지를 위한 Dual Branch Anomaly Detection 시스템에 대한 선행 특허 조사 결과를 종합적으로 정리한 것입니다. 조사 대상 특허와의 기술적 유사점 및 차이점을 분석하고, 본 과제의 독창성과 특허 가능성을 평가합니다.

## 2. 본 과제 기술 구성 요소

### 2.1 기술 구성 요소 개요

본 과제는 다음과 같은 4가지 핵심 기술 요소로 구성됩니다:

**A. Autoencoder 기반 Reconstruction Branch**
- 정상 데이터만으로 학습된 Autoencoder를 통한 재구성 기반 이상 탐지
- 재구성 오차를 이상 스코어로 변환

**B. Feature Attention Branch (CBAM 변형)**
- Latent feature에 Channel/Spatial Attention을 적용하여 이상 특징 추출
- CAM (Channel Attention Map)과 SAM (Spatial Attention Map) 동시 활용

**C. ADRM2D (Adaptive Remapping) + 비선형 결합**
- 학습 가능한 alpha/beta 파라미터 맵을 사용한 비선형 remapping
- Outlier suppression 메커니즘 포함
- Dual Branch 스코어를 비선형적으로 결합

**D. MVTec 스타일 데이터셋 + ROI 마스킹**
- CheXmask 기반 ROI 마스킹 통합
- Balanced Data Loader를 통한 데이터 불균형 해결
- FPR 기반 Threshold 자동 선택

## 2. 본 과제 기술 구성 요소별 비교 분석

### 2.1 A. Autoencoder 기반 Reconstruction Branch

#### 2.1.1 유사점

**US10849585B1 (Siemens)**
- Autoencoder 기반 재구성(reconstruction) 방식을 사용하여 이상을 탐지
- 정상 이미지로 학습된 모델이 이상 이미지에서 높은 재구성 오차를 보임
- 재구성 오차를 이상 스코어로 활용

**CN111242897A 및 기타 특허**
- 대부분의 의료 영상 이상 탐지 특허에서 Autoencoder 기반 재구성 방식을 기본 구조로 사용
- 정상 데이터만으로 학습하는 비지도 학습 접근법

**본 과제**
- `SimpleAE` 모델을 통한 재구성 기반 이상 탐지
- 정상 데이터만으로 학습하여 재구성 오차를 이상 스코어로 변환
- `s_recon = 1.0 - torch.exp(-se)` 형태로 재구성 오차를 정규화

#### 2.1.2 차이점

**US10849585B1 (Siemens)**
- **파라미터화된 X-ray 이미지**: Topogram 예측 및 파라미터화를 통한 이상 탐지
- **표면 데이터 기반**: 환자 표면 데이터로부터 내부 마커를 예측하는 방식
- **CT 스캔 최적화**: 이상 탐지뿐만 아니라 CT 스캔 파라미터 최적화에 초점

**본 과제의 차별점**
1. **Stability Penalty 메커니즘**: 
   - 학습 중 재구성 오차 분포를 안정화하는 `stability_penalty` 함수 도입
   - `stability_target` (기본값 0.05)로 스코어 평균을 수렴시켜 장기적 안정성 확보
   - 코드: `stability_penalty(residual, args.stability_target)`

2. **Huber Loss 적용**:
   - 재구성 손실에 Huber Loss를 적용하여 outlier에 robust한 학습
   - `HuberLoss(delta=0.1)` 사용

3. **ROI 마스킹 통합**:
   - CheXmask 기반 ROI 마스킹을 전처리 단계에서 통합
   - 폐/심장 영역 외부를 마스킹하여 노이즈 제거

---

### 2.2 B. Feature Attention Branch (CBAM 변형)

#### 2.2.1 유사점

**기존 특허들**
- Attention 메커니즘을 사용한 이상 특징 추출
- Channel Attention과 Spatial Attention을 결합한 CBAM 스타일 구조
- Latent feature에서 이상 특징을 강조하는 방식

**본 과제**
- `CSAD2D` 모듈: Channel Attention (`CAM2D`) + Spatial Attention (`SAM2D`) 결합
- Latent feature `z`에 attention을 적용하여 이상 특징 `s_feat` 추출
- CAM과 SAM을 동시에 활용하여 해석 가능한 attention map 생성

#### 2.2.2 차이점

**기존 특허들**
- Attention을 단일 경로로 사용하거나, 분류 모델의 보조 기능으로 활용
- Attention 결과를 직접 이상 스코어로 사용하는 경우가 많음

**본 과제의 차별점**
1. **독립적인 Dual Branch 구조**:
   - Reconstruction Branch와 Feature Attention Branch를 병렬로 구성
   - 각 branch가 독립적으로 학습되며, 이후 ADRM2D에서 비선형 결합
   - 코드: `ImageAnomalyDual` 클래스에서 두 branch를 병렬 처리

2. **Latent Grid 기반 Attention**:
   - Image-level이 아닌 latent feature map (`z: [B,C,h,w]`)에 attention 적용
   - 이후 `F.interpolate`로 image size로 업샘플링하여 해상도 유지

3. **CAM/SAM 동시 활용**:
   - Channel Attention Map (CAM)과 Spatial Attention Map (SAM)을 모두 반환
   - 해석 가능성(interpretability) 향상: `return_intermediate=True` 시 CAM/SAM 시각화 가능

---

### 2.3 C. ADRM2D (Adaptive Remapping) + 비선형 결합

#### 2.3.1 유사점

**기존 특허들**
- 여러 스코어를 결합하여 최종 이상 스코어를 생성하는 하이브리드 접근법
- 가중합(weighted sum) 또는 단순 곱셈을 통한 스코어 결합

**본 과제**
- Reconstruction 스코어(`s_recon`)와 Feature 스코어(`s_feat`)를 결합
- 최종 이상 스코어 생성

#### 2.3.2 차이점

**기존 특허들**
- 고정된 가중치를 사용한 선형 결합
- 단순 평균 또는 가중 평균 사용
- 예: `final_score = w1 * score1 + w2 * score2`

**본 과제의 차별점**
1. **ADRM2D (Adaptive Remapping)**:
   - 학습 가능한 `alpha`/`beta` 파라미터 맵을 사용한 비선형 remapping
   - 공식: `x1 = s + a * (s^2 - s)`, `x2 = x1 + b * (x1^2 - x1)`
   - 각 픽셀마다 다른 remapping 파라미터 적용 (spatially adaptive)
   - 코드: `ADRM2D` 클래스의 `forward` 메서드

2. **Outlier Suppression 메커니즘**:
   - Percentile 기반 outlier clipping: `suppression_percentile=99.5`
   - 각 배치별로 상위 percentile을 제한하여 극단값 억제
   - 코드: `_suppress_outliers` 메서드

3. **DualBranchFusion 모듈**:
   - ADRM2D 출력을 추가로 정제하는 fusion 모듈
   - Learnable gating을 통한 adaptive fusion
   - 코드: `DualBranchFusion` 클래스

4. **CAM/SAM 동시 활용**:
   - 최종 스코어 생성 시 CAM과 SAM을 모두 활용
   - 해석 가능한 attention map과 anomaly map을 동시에 제공

---

### 2.4 D. MVTec 스타일 데이터셋 + ROI 마스킹

#### 2.4.1 유사점

**기존 특허들**
- 의료 영상 전처리 및 데이터셋 구성
- ROI (Region of Interest) 추출 및 마스킹
- 정상/이상 샘플 비율 제어

**본 과제**
- MVTec AD 형식의 데이터셋 구조 사용
- ROI 마스킹을 통한 전처리
- 정상/이상 샘플 비율 제어

#### 2.4.2 차이점

**기존 특허들**
- 전처리와 학습/평가가 분리된 파이프라인
- 데이터 불균형 문제를 별도로 처리하지 않음
- Threshold 선택을 수동 또는 단순 통계 기반으로 수행

**본 과제의 차별점**
1. **통합된 자동화 파이프라인**:
   - ROI 마스킹 → MVTec 변환 → 학습 → 추론 → 평가까지 일괄 자동화
   - `roi_mask_pipeline.py` → `convert_chexpert_to_mvtec.py` → `main.py` 연계
   - 코드: `GUIDE.md`에 전체 파이프라인 명령어 제공

2. **Balanced Data Loader**:
   - 학습 및 평가 시 정상/이상 샘플 비율을 동일하게 맞추는 `get_balanced_train_loader` / `get_balanced_test_loader`
   - 데이터 불균형 문제를 근본적으로 해결
   - 코드: `environment.py`의 balanced loader 구현

3. **FPR 기반 Threshold 선택**:
   - `--target_fpr` 옵션으로 목표 False Positive Rate에 맞춰 threshold 자동 선택
   - 실제 운영 환경 요구사항(FPR 5% 등)에 맞춘 threshold 산정
   - 코드: `eval_ae_scores.py`, `eval_dual_scores.py`의 FPR 기반 thresholding

4. **CheXmask 통합**:
   - CheXmask-Database 기반 HybridGNet 모델을 통한 정확한 폐/심장 ROI 마스킹
   - PA (Posterior-Anterior) 뷰 필터링과 ROI 마스킹을 동시에 수행

### 3.2 기술적 유사점 (공통 기술 요소)

1. **Autoencoder 기반 재구성**: 대부분의 특허에서 사용하는 기본 구조
2. **Attention 메커니즘**: CBAM 스타일의 Channel/Spatial Attention 활용
3. **하이브리드 스코어 결합**: 여러 스코어를 결합하여 최종 판단
4. **ROI 마스킹**: 의료 영상에서 관심 영역 추출 및 전처리
5. **비지도 학습**: 정상 데이터만으로 학습하는 접근법

### 3.3 기술적 차별점 (본 과제의 독창성)

#### 3.2.1 학습 단계 차별점

1. **Stability Penalty 메커니즘**:
   - 학습 중 스코어 분포를 안정화하여 장기적 성능 향상
   - `stability_target`으로 스코어 평균을 목표 값으로 수렴
   - 기존 특허에서는 학습 단계에서의 스코어 안정화를 다루지 않음

2. **Robust Loss Functions**:
   - Huber Loss + RobustScoreLoss 조합
   - Outlier에 robust한 학습으로 일반화 성능 향상

3. **Balanced Training**:
   - 학습 단계부터 정상/이상 샘플 비율을 균형화
   - 데이터 불균형 문제를 근본적으로 해결

#### 3.2.2 모델 구조 차별점

1. **ADRM2D (Adaptive Remapping)**:
   - 학습 가능한 `alpha`/`beta` 파라미터 맵을 사용한 비선형 remapping
   - Spatially adaptive한 스코어 변환
   - 기존 특허의 고정 가중치 선형 결합과 차별화

2. **Outlier Suppression**:
   - Percentile 기반 outlier clipping
   - 극단값에 robust한 스코어 계산

3. **Dual Branch 병렬 구조**:
   - Reconstruction과 Feature Attention을 독립적인 branch로 구성
   - 각 branch의 특성을 최대한 활용한 후 결합

#### 3.2.3 평가/운영 단계 차별점

1. **FPR 기반 Threshold 선택**:
   - 실제 운영 환경 요구사항에 맞춘 threshold 자동 선택
   - 기존 특허의 수동 또는 단순 통계 기반 threshold와 차별화

2. **Balanced Evaluation**:
   - 평가 시에도 정상/이상 샘플 비율을 균형화
   - 불균형 데이터셋에서도 정확한 성능 평가 가능

3. **통합 자동화 파이프라인**:
   - 전처리부터 평가까지 일괄 자동화
   - 재현성(reproducibility) 향상

### 4.2 기타 특허 vs 본 과제

#### 4.2.1 스코어 결합 방식 비교

| 특허/연구 | 스코어 결합 방식 | 본 과제 | 차별화 |
|---------|--------------|--------|--------|
| 일반 특허 | 선형 가중합: `w1*s1 + w2*s2` | ADRM2D: `s + a*(s²-s)`, 이중 적용 | ✅ 비선형 |
| 일반 특허 | 고정 가중치 | 학습 가능한 alpha/beta 맵 | ✅ Adaptive |
| 일반 특허 | 단순 평균/곱셈 | Outlier suppression + Fusion | ✅ Robust |

#### 4.2.2 학습/평가 파이프라인 비교

| 특허/연구 | 데이터 불균형 처리 | Threshold 선택 | 본 과제 | 차별화 |
|---------|----------------|--------------|--------|--------|
| 일반 특허 | 별도 처리 없음 | 수동 또는 단순 통계 | Balanced Loader | ✅ 근본 해결 |
| 일반 특허 | 평가 단계만 처리 | F1-optimal 등 | FPR 기반 자동 선택 | ✅ 운영 맞춤 |
| 일반 특허 | 분리된 파이프라인 | 수동 실행 | 통합 자동화 | ✅ 재현성 |

## 5. 특허 회피 전략 및 독창성 분석

### 5.1 특허 회피 전략

#### 5.1.1 US10849585B1 (Siemens) 회피 포인트

1. **파라미터화된 X-ray 이미지 미사용**:
   - 본 과제는 일반 X-ray 이미지를 직접 사용하며, Topogram 예측이나 파라미터화 과정을 포함하지 않음
   - 표면 데이터 기반 예측 방식을 사용하지 않음

2. **CT 스캔 최적화 목적 없음**:
   - 이상 탐지에만 초점을 맞추며, CT 스캔 파라미터 최적화는 목적이 아님

3. **차별화된 기술 요소**:
   - ADRM2D 비선형 remapping (기존 특허는 선형 결합)
   - Stability Penalty 메커니즘 (기존 특허에 없음)

#### 5.1.2 기타 특허 회피 포인트

1. **ADRM2D의 비선형 remapping**:
   - 기존 특허의 선형 가중합(`w1*s1 + w2*s2`)과 차별화된 비선형 결합 방식
   - 학습 가능한 alpha/beta 파라미터 맵 사용

2. **Stability Penalty 메커니즘**:
   - 학습 단계에서의 스코어 안정화는 기존 특허에서 다루지 않는 독창적 요소
   - 목표 스코어 평균으로의 수렴 메커니즘

3. **Balanced Loader + FPR 기반 Threshold**:
   - 데이터 불균형 문제 해결과 운영 환경 맞춤 threshold 선택은 기존 특허와 차별화

### 5.2 기술적 독창성 평가

#### 5.2.1 핵심 독창 기술 요소

| 기술 요소 | 독창성 수준 | 기존 특허 대비 |
|---------|-----------|-------------|
| **ADRM2D** | ⭐⭐⭐⭐⭐ | 비선형 remapping, 학습 가능한 파라미터 맵 |
| **Stability Penalty** | ⭐⭐⭐⭐⭐ | 학습 단계 스코어 안정화 (기존 특허 없음) |
| **Dual Branch 병렬 구조** | ⭐⭐⭐⭐ | 독립적 학습 후 비선형 결합 |
| **Outlier Suppression** | ⭐⭐⭐⭐ | Percentile 기반 clipping |
| **Balanced Loader** | ⭐⭐⭐ | 데이터 불균형 근본 해결 |
| **FPR 기반 Threshold** | ⭐⭐⭐ | 운영 환경 맞춤 자동 선택 |

#### 5.2.2 종합 독창성 평가

본 과제는 기존 특허들이 사용하는 기본 기술 요소(Autoencoder, Attention, 하이브리드 결합)를 활용하되, 다음과 같은 독창적 개선을 제시합니다:

1. **ADRM2D**: 학습 가능한 비선형 remapping을 통한 adaptive 스코어 결합
2. **Stability Penalty**: 학습 단계에서의 스코어 분포 안정화
3. **Balanced Loader + FPR 기반 Threshold**: 데이터 불균형 문제 해결 및 운영 환경 맞춤
4. **통합 자동화 파이프라인**: 전처리부터 평가까지 일괄 자동화

### 5.3 특허 가능성 평가

**결론**: 본 과제의 핵심 기술 요소인 **ADRM2D (Adaptive Remapping with Dual Branch)**와 **Stability Penalty 메커니즘**은 기존 특허와 차별화된 독창적 기술로 판단되며, **특허 출원 가능성이 높습니다**.

**근거**:
1. ADRM2D는 기존 특허의 선형 가중합과 본질적으로 다른 비선형 remapping 방식
2. Stability Penalty는 기존 특허에서 다루지 않는 학습 단계 스코어 안정화 메커니즘
3. 통합된 자동화 파이프라인은 실용성 측면에서 차별화

### 5.4 권리 범위 제안

#### 5.4.1 독립항 제안

**청구항 1**: 학습 가능한 alpha/beta 파라미터 맵을 사용하여 두 이상 탐지 스코어를 비선형적으로 remapping하는 ADRM2D(Adaptive Remapping) 모듈을 포함하는 의료 영상 이상 탐지 방법
