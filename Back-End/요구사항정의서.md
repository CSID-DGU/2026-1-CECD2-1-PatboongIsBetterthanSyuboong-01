# BitComputer 시스템 요구기능 정의서

## 1. 개요

### 1.1 시스템 목적
BitComputer는 의료진료 기록 관리 시스템으로, 환자 정보 관리, 진료 기록 작성 및 관리, 질병/진단 코드 검색, 대기 목록 관리 등의 기능을 제공합니다.

### 1.2 문서 목적
본 문서는 BitComputer 시스템이 제공해야 하는 기능적 요구사항을 정의합니다. 각 기능은 고유 ID를 부여받으며, 카테고리별로 분류되어 있습니다.

---

## 2. 사용자 인증 및 관리

| ID | 정의 | 상세설명 |
|---|---|---|
| **REQ-USER-001** | 사용자 등록 | 새로운 직원을 시스템에 등록하는 기능. 사용자명, 비밀번호 등 필수 정보를 입력받아 시스템에 저장하며, 사용자명 중복 여부를 검사하여 중복된 경우 등록을 거부합니다. 등록 성공 시 해당 직원은 시스템에 로그인할 수 있는 권한을 획득합니다. |
| **REQ-USER-002** | 사용자 로그인 | 등록된 직원이 사용자명과 비밀번호를 입력하여 시스템에 인증하는 기능. 인증 성공 시 JWT 기반의 Access Token과 Refresh Token을 발급하며, 이후 API 호출 시 이 토큰을 사용하여 인증 상태를 유지합니다. 인증 실패 시 로그인을 거부합니다. |
| **REQ-USER-003** | 사용자 로그아웃 | 현재 로그인한 사용자가 시스템에서 로그아웃하는 기능. 로그아웃 시 발급받은 JWT 토큰을 블랙리스트에 등록하여 더 이상 사용할 수 없도록 무효화합니다. 이를 통해 토큰 탈취 시에도 보안을 유지할 수 있습니다. |
| **REQ-USER-004** | 현재 사용자 역할 조회 | 현재 로그인한 사용자의 역할(Role) 정보를 조회하는 기능. JWT 토큰에서 사용자 정보를 추출하여 해당 사용자의 역할을 반환합니다. 역할 정보는 화면에서 권한에 따른 UI 표시나 기능 제어에 활용됩니다. |

## 3. 권한 관리

| ID | 정의 | 상세설명 |
|---|---|---|
| **REQ-AUTH-001** | 사용자 역할 변경 | SUPER_USER 권한을 가진 사용자만 다른 직원의 역할을 변경할 수 있는 기능. 대상 직원의 ID와 변경할 역할을 입력받아 역할을 업데이트합니다. 역할 변경 시 현재 사용자가 SUPER_USER인지 검증하며, 유효하지 않은 역할 값이 입력된 경우 변경을 거부합니다. |
| **REQ-AUTH-002** | 전체 사용자 목록 조회 | 시스템에 등록된 모든 직원의 목록을 조회하는 기능. 직원의 기본 정보(사용자명, 역할 등)를 포함한 목록을 반환하며, 관리자가 전체 직원 현황을 파악하는 데 사용됩니다. |

## 4. 환자 관리

| ID | 정의 | 상세설명 |
|---|---|---|
| **REQ-PATIENT-001** | 환자 등록 | 새로운 환자를 시스템에 등록하는 기능. 환자의 기본 정보(이름, 주민등록번호, 연락처 등)를 입력받아 데이터베이스에 저장하고, 시스템에서 사용할 고유한 환자 ID를 생성하여 반환합니다. 환자 정보는 진료 기록 작성 시 필수적으로 사용됩니다. |
| **REQ-PATIENT-002** | 환자 정보 조회 (ID) | 환자 ID를 기준으로 특정 환자의 상세 정보를 조회하는 기능. 환자 ID를 입력받아 해당 환자의 모든 정보를 반환합니다. 존재하지 않는 환자 ID인 경우 조회 결과가 없음을 반환합니다. |
| **REQ-PATIENT-003** | 환자 정보 조회 (주민등록번호) | 주민등록번호를 기준으로 환자 정보를 조회하는 기능. 주민등록번호를 입력받아 해당 환자의 정보를 검색하여 반환합니다. 중복 등록 방지나 기존 환자 확인 시 사용됩니다. |
| **REQ-PATIENT-004** | 전체 환자 목록 조회 | 시스템에 등록된 모든 환자의 목록을 조회하는 기능. 환자들의 기본 정보를 포함한 목록을 반환하며, 환자 검색이나 전체 현황 파악에 사용됩니다. 대량의 데이터인 경우 성능 최적화가 필요할 수 있습니다. |

## 5. 진료 기록 관리

| ID | 정의 | 상세설명 |
|---|---|---|
| **REQ-HISTORY-001** | 진료 기록 작성 | 새로운 진료 기록을 작성하는 기능. 환자 ID, 진료 일시, 진료 내용, 담당 직원 정보 등을 입력받아 진료 기록을 생성합니다. 진료 기록은 환자의 진료 이력을 관리하는 핵심 데이터이며, 이후 수정 및 조회의 기준이 됩니다. |
| **REQ-HISTORY-002** | 진료 기록 수정 | 기존에 작성된 진료 기록의 내용을 수정하는 기능. 진료 기록 ID를 기준으로 해당 기록을 찾아 입력받은 정보로 업데이트합니다. 진료 기록이 존재하지 않는 경우 수정을 거부하며, 수정된 내용은 즉시 반영됩니다. |
| **REQ-HISTORY-003** | 진료 기록 검색 (환자 기준) | 특정 환자의 진료 기록을 검색하는 기능. 환자 ID를 필수 조건으로 하며, 선택적으로 시작 날짜와 종료 날짜를 지정하여 특정 기간의 진료 기록만 조회할 수 있습니다. 날짜 범위가 유효하지 않은 경우(시작일이 종료일보다 늦은 경우) 검색을 거부합니다. 검색 결과는 진료 기록 목록과 관련 통계 정보를 포함합니다. |
| **REQ-HISTORY-004** | 진료 기록에 질병 정보 설정 | 특정 진료 기록에 질병 목록을 일괄 설정하는 기능. 진료 기록 ID와 질병 정보 목록을 입력받아 해당 진료 기록의 기존 질병 정보를 모두 삭제하고 새로운 질병 목록으로 교체합니다. 각 질병 정보는 질병 코드, 질병명, 정도(degree) 등을 포함할 수 있습니다. |
| **REQ-HISTORY-005** | 진료 기록의 질병 정보 조회 | 특정 진료 기록에 등록된 질병 목록을 조회하는 기능. 진료 기록 ID를 입력받아 해당 진료 기록에 연결된 모든 질병 정보를 반환합니다. 질병 정보는 질병 코드, 질병명, 정도 등의 상세 정보를 포함합니다. |
| **REQ-HISTORY-006** | 진료 기록에 질병 추가 | 기존 진료 기록에 새로운 질병을 추가하는 기능. 진료 기록 ID와 질병 ID를 입력받아 해당 질병을 진료 기록에 연결합니다. 선택적으로 질병의 정도(degree) 정보를 함께 저장할 수 있습니다. 질병 정보는 질병 코드 마스터에서 조회하여 자동으로 채워집니다. |
| **REQ-HISTORY-007** | 진료 기록에 진단 정보 설정 | 특정 진료 기록에 진단 목록을 일괄 설정하는 기능. 진료 기록 ID와 진단 정보 목록을 입력받아 해당 진료 기록의 기존 진단 정보를 모두 삭제하고 새로운 진단 목록으로 교체합니다. 각 진단 정보는 진단 코드와 진단명을 포함합니다. |
| **REQ-HISTORY-008** | 진료 기록의 진단 정보 조회 | 특정 진료 기록에 등록된 진단 목록을 조회하는 기능. 진료 기록 ID를 입력받아 해당 진료 기록에 연결된 모든 진단 정보를 반환합니다. 진단 정보는 진단 코드와 진단명 등의 상세 정보를 포함합니다. |
| **REQ-HISTORY-009** | 진료 기록에 진단 추가 | 기존 진료 기록에 새로운 진단을 추가하는 기능. 진료 기록 ID와 진단 ID를 입력받아 해당 진단을 진료 기록에 연결합니다. 진단 정보는 진단 코드 마스터에서 조회하여 자동으로 채워집니다. |

## 6. 질병 코드 관리

| ID | 정의 | 상세설명 |
|---|---|---|
| **REQ-DISEASE-001** | 질병 코드 조회 (ID) | 질병 ID를 기준으로 특정 질병의 상세 정보를 조회하는 기능. 질병 ID를 입력받아 해당 질병의 코드, 이름 등 모든 정보를 반환합니다. 존재하지 않는 질병 ID인 경우 조회 결과가 없음을 반환합니다. |
| **REQ-DISEASE-002** | 질병 코드 검색 | 다양한 조건으로 질병 코드를 검색하는 기능. 통합 검색어(query), 질병 코드(code), 질병명(name) 중 하나 이상의 조건을 사용하여 검색할 수 있으며, 검색 결과는 페이지네이션을 지원합니다. 기본 페이지 크기는 50개이며, 페이지 번호와 크기를 조정하여 대량의 데이터를 효율적으로 조회할 수 있습니다. 검색 결과는 질병 코드와 이름을 포함한 목록으로 반환됩니다. |
| **REQ-DISEASE-003** | 질병 코드 일괄 등록 (Excel) | Excel 파일을 통해 질병 코드를 일괄로 등록하는 기능. 표준화된 Excel 파일 형식을 읽어서 질병 코드 정보를 데이터베이스에 저장합니다. 대량의 질병 코드를 한 번에 등록할 때 사용되며, 등록된 레코드 수를 반환합니다. 파일 형식 오류나 데이터 오류 시 적절한 오류 처리가 필요합니다. |

## 8. 대기 목록 관리

| ID | 정의 | 상세설명 |
|---|---|---|
| **REQ-WAITING-001** | 대기 목록 등록 | 환자를 대기 목록에 등록하는 기능. 환자 ID를 입력받아 해당 환자를 대기 목록에 추가합니다. 대기 등록 시 대기 상태가 설정되며, 등록 성공 시 JWT 토큰을 발급하여 이후 진료 진행 시 사용할 수 있도록 합니다. 환자 ID가 유효하지 않은 경우 등록을 거부합니다. |
| **REQ-WAITING-002** | 대기 목록 조회 | 현재 대기 중인 모든 환자의 목록을 조회하는 기능. 대기 상태에 있는 환자들의 정보를 반환하며, 대기 순서, 환자 정보 등을 포함합니다. 빈 목록인 경우에도 정상 응답으로 처리됩니다. |
| **REQ-WAITING-003** | 대기 완료 처리 | 특정 환자의 대기 상태를 완료로 변경하는 기능. 환자 ID를 입력받아 해당 환자의 대기 상태를 완료로 업데이트합니다. 완료 처리 시 JWT 토큰을 발급하여 진료 기록 작성 등의 후속 작업에 사용할 수 있도록 합니다. 환자 ID가 유효하지 않거나 대기 목록에 없는 경우 처리를 거부합니다. |
| **REQ-WAITING-004** | 대기 보류 처리 | 특정 환자의 대기 상태를 보류로 변경하는 기능. 환자 ID를 입력받아 해당 환자의 대기 상태를 보류로 업데이트합니다. 보류 상태는 일시적으로 진료를 중단하거나 나중에 다시 진행할 때 사용됩니다. 보류 처리 시 JWT 토큰을 발급하여 필요 시 상태를 다시 변경할 수 있도록 합니다. 환자 ID가 유효하지 않거나 대기 목록에 없는 경우 처리를 거부합니다. |

## 9. 시스템 관리

| ID | 정의 | 상세설명 |
|---|---|---|
| **REQ-SYSTEM-001** | 데이터베이스 연결 상태 확인 | 시스템이 사용하는 데이터베이스(MySQL, Redis)의 연결 상태를 확인하는 기능. MySQL과 Redis 각각에 대해 연결 테스트를 수행하고, 연결 상태(UP/DOWN), 연결 정보(URL, 호스트, 포트 등) 또는 오류 메시지를 반환합니다. 시스템 모니터링이나 장애 진단 시 사용됩니다. |

## 10. 기능 요약

### 10.1 기능 통계
- **총 기능 수**: 30개
- **사용자 인증 및 관리**: 4개
- **권한 관리**: 2개
- **환자 관리**: 4개
- **진료 기록 관리**: 9개
- **질병 코드 관리**: 3개
- **진단 코드 관리**: 3개
- **대기 목록 관리**: 4개
- **시스템 관리**: 1개

### 10.2 우선순위 분류
- **필수 기능 (P0)**: 사용자 인증, 환자 관리, 진료 기록 작성/조회
- **중요 기능 (P1)**: 진료 기록 수정, 질병/진단 코드 검색, 대기 목록 관리
- **보조 기능 (P2)**: 권한 관리, 시스템 상태 확인, Excel 일괄 등록

---

## 11. 비기능 요구사항

### 11.1 보안 요구사항
- 모든 API는 JWT 토큰 기반 인증을 지원해야 합니다.
- 비밀번호는 암호화되어 저장되어야 합니다.
- SUPER_USER만 다른 사용자의 역할을 변경할 수 있어야 합니다.
- 로그아웃 시 토큰은 블랙리스트에 등록되어 무효화되어야 합니다.

### 11.2 성능 요구사항
- 질병/진단 코드 검색은 페이지네이션을 지원하여 대량 데이터 조회 시 성능을 보장해야 합니다.
- 진료 기록 검색은 날짜 범위 조건을 활용하여 효율적인 조회가 가능해야 합니다.

### 11.3 데이터 무결성 요구사항
- 환자 정보는 중복 등록을 방지해야 합니다.
- 진료 기록 수정 시 기존 데이터의 무결성을 보장해야 합니다.
- 질병/진단 코드는 마스터 데이터로 관리되어야 합니다.

## 12. 용어 정의

| 용어 | 정의 |
|---|---|
| **진료 기록 (History)** | 환자의 진료 이력을 기록한 데이터로, 진료 일시, 내용, 담당 직원 등의 정보를 포함합니다. |
| **질병 코드 (Disease)** | 표준화된 질병 분류 코드로, 진료 기록에 질병 정보를 기록할 때 사용됩니다. |
| **진단 코드 (Diagnose)** | 표준화된 진단 분류 코드로, 진료 기록에 진단 정보를 기록할 때 사용됩니다. |
| **대기 목록 (Waiting)** | 진료를 대기 중인 환자들의 목록으로, 대기 상태를 관리합니다. |
| **JWT 토큰** | JSON Web Token으로, 사용자 인증 및 권한 관리를 위해 사용되는 토큰입니다. |
| **SUPER_USER** | 시스템의 최고 권한을 가진 사용자로, 다른 사용자의 역할을 변경할 수 있습니다. |
